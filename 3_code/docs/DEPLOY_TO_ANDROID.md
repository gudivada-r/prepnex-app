# Deploying Student Success Navigator to Android

This guide explains how to build the APK/AAB file and deploy it to Android devices (Testing) or the Google Play Store (Production).

## Prerequisites
1.  **React Build**: Recent `npm run build` of the frontend.
2.  **Capacitor**: `npx cap sync android` must be run.
3.  **Android Studio**: Installed with Android SDK and Java JDK (usually included).
4.  **Google Play Developer Account**: Required for Store deployment (~$25 one-time fee).

---

## 🚀 Quick Start (Automated Script)
We have prepared a PowerShell script to handle the build and sync automatically.

1. Open Terminal in the project root.
2. Run:
   ```powershell
   ./build_android.ps1
   ```
3. This will:
   * Rebuild the Vite/React app.
   * Sync the assets to the `android/` folder.
   * Attempt to build a Debug APK using Gradle.

If the script succeeds, your debug APK is located at:
`frontend/android/app/build/outputs/apk/debug/app-debug.apk`

---

## 📱 Option A: Deploy to Test Device (Local)
*Best for development testing.*

1. **Enable Developer Mode** on your Android phone:
   * Go to *Settings > About Phone*.
   * Tap *Build Number* 7 times until it says "You are a developer".
   * Go to *System > Developer Options* and enable **USB Debugging**.
2. **Connect Phone** to your PC via USB.
3. **Run via Android Studio**:
   * Open Android Studio.
   * Select `File > Open` and choose `c:\Projects\AA\SS_12_26\frontend\android`.
   * Wait for Gradle sync to finish.
   * Select your connected phone in the device dropdown (top toolbar).
   * Click the green **Run (Play icon)** button.
   * The app will install and open on your phone.

OR 

4. **Install APK Manually**:
   * Transfer `app-debug.apk` (generated by the script) to your phone via USB or Google Drive.
   * Open the file on your phone and tap "Install".

## 🌍 Option B: Internal Testing (Google Play)
*Best for sharing with beta testers.*

### 1. Versioning
Open `frontend/android/app/build.gradle` and increment the version:
```gradle
defaultConfig {
    versionCode 2  // Increment this by 1 for every upload
    versionName "1.1"
}
```

### 2. Signing
You cannot upload a Debug APK to Google Play. You must sign it.
1. In Android Studio, go to **Build > Generate Signed Bundle / APK**.
2. Select **Android App Bundle (.aab)** (Preferred) or APK.
3. Create a new **Key Store** path (save this `jks` file safely! If you lose it, you can't update the app).
4. Enter passwords and alias details.
5. Select **Release** build variant.
6. Click **Finish**.

### 3. Uploading
1. Go to [Google Play Console](https://play.google.com/console).
2. Create App -> "Student Success Navigator".
3. Go to **Testing > Internal testing**.
4. Create new release.
5. Upload the `.aab` file generated in step 2.
6. Add tester email addresses (they will receive a download link).

## Common Issues

**1. "Unknown Command: npx"**
Ensure Node.js is installed and added to your PATH.

**2. "SDK Location not found"**
Create a `local.properties` file in `frontend/android/` with:
`sdk.dir=C:\\Users\\YOUR_USER\\AppData\\Local\\Android\\Sdk`

**3. App Crashes on Launch (API Error)**
The Android emulator uses `10.0.2.2` to access localhost. A real device cannot access `localhost`.
*   **Fix**: Update `frontend/src/api.js` or `capacitor.config.json` to point to your **Production URL** (e.g., Vercel/Render) instead of `localhost:8000`.
